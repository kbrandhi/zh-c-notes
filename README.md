# zh-c-notes

## 项目简介

`zh-c-notes` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为通用型的页面存放与版本管理空间。  
每个 HTML 页面可视为独立的笔记、工具页面或轻量级前端演示，便于通过 GitHub Pages 等方式直接访问。

## 目录结构

```
zh-c-notes/
├── index.html          # 首页或目录导航（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── page1.html
│   └── page2.html
└── assets/             # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/` 目录用于存放各个独立的 HTML 页面文件。
- `assets/` 目录用于存放页面共享的样式、脚本及媒体文件。
- 根目录下的 `index.html` 可作为导航页或默认入口。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立归档。  
- 页面之间无强依赖关系，可单独访问。
- 文件名应体现页面内容或用途，建议使用英文或拼音命名。
- 如需引用外部资源（如字体库、CDN 库），请注明来源或使用相对路径。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面或改进现有页面。
- 请保持页面结构清晰，避免引入恶意代码或外部跟踪脚本。
- 资源文件请尽量放置于 `assets/` 目录下，便于统一管理。
- 提交前请确认页面在本地或测试环境中能正常渲染。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 通过 GitHub Pages 或其他静态托管服务部署即可访问。
